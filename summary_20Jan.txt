1. Table Structure (The Grid)
Tables are used to align elements in rows and columns. This ensures your labels and input boxes stay perfectly lined up.

Tags: <table>, <tr> (row), <td> (cell/column).

Syntax: <table><tr><td>Content</td></tr></table>

Use: Creating a structured layout where the left column is for labels and the right column is for inputs.

2. Table Attributes (The Look)
These attributes control the spacing and the "rectangle" border around your form.

Tags/Attributes: * border="1": Draws the outer rectangle and cell lines.

cellpadding="10": Adds space inside cells so text doesn't touch the borders.

align="center": Moves the entire table to the middle of the page.

Use: Making the form look organized and easy to read.

3. Column Spanning (Merging)
This allows one cell to "stretch" across multiple columns.

Attribute: colspan="x" (where x is the number of columns to merge).

Syntax: <td colspan="2">...</td>

Use: Centering the Submit and Reset buttons across the bottom of the entire form.

4. Inline Styling (CSS in HTML)
Adding styles directly to tags to change their appearance without a separate stylesheet.

Attribute: style="..."

Syntax: <small style="color: red;"> or <input style="background-color: green;">

Use: Making "Notes" red for attention and making buttons colorful (Green for Submit, Red for Reset).

5. Semantic & Formatting Tags

Tags that add meaning or specific looks to your text.

Tags: <b>: Bold text for labels.

<small>: Smaller text for notes.

<br>: A simple line break to move to the next line.

Use: Highlighting important information and controlling text flow

Text

Syntax: <input type="text" name="username">

Use Case: Short entries like First Name, Last Name, or City.

Password

Syntax: <input type="password" name="pwd">

Use Case: Security fields; it masks characters as dots or asterisks.

Email

Syntax: <input type="email" name="user_email">

Use Case: Newsletter signups or contact forms; it automatically checks for an @ symbol.

Number

Syntax: <input type="number" name="age" min="1" max="100">

Use Case: Age, quantity, or years of experience. It provides up/down arrows.

Date

Syntax: <input type="date" name="birthday">

Use Case: Date of Birth or appointment scheduling; it opens a built-in calendar.

File

Syntax: <input type="file" name="upload">

Use Case: Uploading photographs, resumes, or ID proofs.

Checkbox

Syntax: <input type="checkbox" name="hobby" value="sports">

Use Case: Situations where a user can select multiple options (e.g., "Select your interests").

Radio

Syntax: <input type="radio" name="gender" value="male">

Use Case: Situations where a user can select only one option (e.g., Gender or Yes/No).

Color

Syntax: <input type="color" name="favcolor">

Use Case: Preference settings; it opens a color picker interface.

Range

Syntax: <input type="range" name="volume" min="0" max="10">

Use Case: Volume sliders, price filters, or "satisfaction" scales.

Submit

Syntax: <input type="submit" value="Send Data">

Use Case: The final button that sends the form information to the destination.

Reset

Syntax: <input type="reset" value="Clear Form">

Use Case: A button that clears all the text and selections the user just made.
